## 1. setuptools-scm Integration

### Overview
setuptools-scm extracts Python package versions from Git metadata instead of declaring them manually. It's ideal for portable applications requiring Git-based semantic versioning.

### Configuration in pyproject.toml

#### Basic Setup (Recommended)
```toml
[build-system]
requires = ["setuptools>=80", "setuptools-scm[simple]>=8"]
build-backend = "setuptools.build_meta"

[project]
name = "pymm"
# Remove any existing version declaration
dynamic = ["version"]

# No [tool.setuptools_scm] section needed for basic usage!
```

#### Advanced Configuration with Version File
```toml
[build-system]
requires = ["setuptools>=80", "setuptools-scm>=8"]
build-backend = "setuptools.build_meta"

[project]
name = "pymm"
dynamic = ["version"]

[tool.setuptools_scm]
# Write version to a Python file for runtime access
version_file = "src/pymm/_version.py"

# Custom Git describe command (optional)
[tool.setuptools_scm.scm.git]
describe_command = "git describe --dirty --tags --long --match v*"
```

### Best Practices for Git-Based Semantic Versioning

#### Tag Format (Critical)
**Always use the "v" prefix for version tags:**
```bash
git tag v1.0.0        # Recommended
git tag v1.1.0
git tag v2.0.0-alpha1
git tag v2.0.0-rc1
```

**Why "v" prefix?**
1. **Clarity:** Immediately obvious that tag represents a version
2. **Convention:** Widely adopted standard
3. **Git compatibility:** Works well with git's tag sorting
4. **Tool compatibility:** Many tools expect "v" prefix

#### Versioning Scheme
setuptools-scm generates versions based on:
1. **Latest tag** (with version number)
2. **Distance** to tag (number of commits since)
3. **Workdir state** (uncommitted changes)

**Version Format Logic:**

| Distance to Tag | Uncommitted Changes | Generated Version |
|-----------------|---------------------|-------------------|
| No (on tag) | No | `{tag}` |
| Yes | No | `{next_version}.dev{distance}+g{hash}` |
| No (on tag) | Yes | `{tag}+dYYYYMMDD` |
| Yes | Yes | `{next_version}.dev{distance}+g{hash}.dYYYYMMDD` |

**Examples:**
- On tag `v1.0.0`: `1.0.0`
- 5 commits after `v1.0.0`: `1.0.1.dev5+g1a2b3c4`
- On tag with changes: `1.0.0+d20260103`

### Embedding Version Information in Built Application

#### Method 1: Via importlib.metadata (Recommended)
```python
# src/pymm/__init__.py
from importlib.metadata import version, PackageNotFoundError

try:
    __version__ = version("pymm")
except PackageNotFoundError:
    __version__ = "unknown"
```

#### Method 2: Via Version File
If using `version_file` configuration:
```python
# src/pymm/__init__.py
try:
    from pymm._version import version as __version__
    from pymm._version import version_tuple
except ImportError:
    __version__ = "unknown"
    version_tuple = (0, 0, 0)
```

**Add _version.py to .gitignore:**
```gitignore
# Generated version file
src/pymm/_version.py
```

### Integration with GitHub Actions for Release Workflows

#### Workflow Patterns

**Pattern 1: Automatic Version on Push**
```yaml
name: Build Release
on:
  push:
    tags:
      - 'v*'  # Triggers on version tags

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # CRITICAL: Get full history for setuptools-scm
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      
      - name: Install build dependencies
        run: |
          python -m pip install --upgrade pip
          pip install build setuptools-scm[simple]
      
      - name: Display version
        run: python -m setuptools_scm
      
      - name: Build package
        run: python -m build
```

**Pattern 2: Version in Build Artifacts**
```yaml
- name: Get version
  id: get_version
  run: |
    echo "version=$(python -m setuptools_scm)" >> $GITHUB_OUTPUT

- name: Create release archive
  run: |
    $version = "${{ steps.get_version.outputs.version }}"
    Compress-Archive -Path dist/* -DestinationPath "pymm-$version-win64.zip"
```

### Portable Python Considerations

#### Challenge: No .git in Portable Distribution
When distributing as portable app, `.git` directory won't be included. Solutions:

**Solution 1: Embed at Build Time**
```yaml
- name: Build with embedded version
  run: |
    # Version gets embedded during build
    python -m build
    # The built package contains version in metadata
```

**Solution 2: Use Version File**
```toml
[tool.setuptools_scm]
version_file = "src/pymm/_version.py"
write_to_template = """
# This file is auto-generated by setuptools-scm

```

**Solution 3: Pretend Version for Docker/Container Builds**
```bash

```

## 2. GitHub Actions for Portable Python Builds

### Workflow Patterns for Embeddable Python

#### Complete Build Workflow Example
```yaml
name: Build Portable Release

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:

env:
  PYTHON_VERSION: '3.11.9'
  APP_NAME: 'pyMM'

jobs:
  build-windows:
    runs-on: windows-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for setuptools-scm
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: ${{ env.PYTHON_VERSION }}
      
      - name: Download embeddable Python
        run: |
          $pythonVersion = "${{ env.PYTHON_VERSION }}"
          $pythonUrl = "https://www.python.org/ftp/python/$pythonVersion/python-$pythonVersion-embed-amd64.zip"
          
          New-Item -ItemType Directory -Force -Path "portable_python"
          Invoke-WebRequest -Uri $pythonUrl -OutFile "python-embed.zip"
          Expand-Archive -Path "python-embed.zip" -DestinationPath "portable_python"
          Remove-Item "python-embed.zip"
      
      - name: Configure embeddable Python
        run: |
          # Enable site-packages
          $pthFile = Get-ChildItem -Path "portable_python" -Filter "*._pth" | Select-Object -First 1
          (Get-Content $pthFile.FullName) -replace '#import site', 'import site' | Set-Content $pthFile.FullName
          
          # Add additional paths
          Add-Content $pthFile.FullName "Lib"
          Add-Content $pthFile.FullName "Lib\site-packages"
      
      - name: Install pip in embeddable Python
        run: |
          Invoke-WebRequest -Uri "https://bootstrap.pypa.io/get-pip.py" -OutFile "get-pip.py"
          .\portable_python\python.exe get-pip.py
          Remove-Item "get-pip.py"
      
      - name: Install dependencies
        run: |
          .\portable_python\python.exe -m pip install --upgrade pip
          .\portable_python\python.exe -m pip install -r requirements.txt --target=".\portable_python\Lib\site-packages"
      
      - name: Freeze dependencies
        run: |
          .\portable_python\python.exe -m pip freeze > requirements-frozen.txt
      
      - name: Copy application files
        run: |
          Copy-Item -Recurse -Path "src\pymm" -Destination "portable_python\Lib\site-packages\pymm"
          Copy-Item -Path "launcher.pyw" -Destination "portable_python\"
          Copy-Item -Path "README.md" -Destination "portable_python\"
          Copy-Item -Path "LICENSE" -Destination "portable_python\"
      
      - name: Get version
        id: get_version
        run: |
          $version = python -m setuptools_scm
          echo "version=$version" >> $env:GITHUB_OUTPUT
          echo "Version: $version"
      
      - name: Create portable archive
        run: |
          $version = "${{ steps.get_version.outputs.version }}"
          $archiveName = "${{ env.APP_NAME }}-$version-win64-portable.zip"
          Compress-Archive -Path "portable_python\*" -DestinationPath $archiveName
          echo "archive_name=$archiveName" >> $env:GITHUB_OUTPUT
        id: create_archive
      
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: ${{ env.APP_NAME }}-windows-portable
          path: ${{ steps.create_archive.outputs.archive_name }}
          retention-days: 30
```

### Dependency Installation and Freezing Strategies

#### Strategy 1: requirements-frozen.txt Pattern
```yaml
- name: Install and freeze dependencies
  run: |
    # Install from requirements.txt
    .\portable_python\python.exe -m pip install -r requirements.txt
    
    # Freeze exact versions (includes transitive dependencies)
    .\portable_python\python.exe -m pip freeze > requirements-frozen.txt
    
    # Include frozen file in release
    Copy-Item requirements-frozen.txt portable_python\
```

**Benefits:**
- Reproducible builds
- Documents exact dependency versions
- Includes transitive dependencies
- Useful for debugging version conflicts

#### Strategy 2: Lock File with pip-tools
```yaml
- name: Install pip-tools
  run: pip install pip-tools

- name: Compile requirements
  run: pip-compile requirements.in --output-file=requirements.txt

- name: Install locked dependencies
  run: |
    .\portable_python\python.exe -m pip install -r requirements.txt
```

#### Strategy 3: Vendoring Critical Dependencies
```yaml
- name: Vendor dependencies
  run: |
    # Install to specific directory
    .\portable_python\python.exe -m pip install -r requirements.txt `
      --target=".\portable_python\vendor"
    
    # Update import paths in launcher
    # Add vendor to sys.path
```

```yaml
- name: Create Release
  uses: softprops/action-gh-release@v1
  if: startsWith(github.ref, 'refs/tags/')
  with:
    files: |
      ${{ env.APP_NAME }}-*-win64-portable.zip
      requirements-frozen.txt

      2. Extract to desired location
      3. Run `launcher.pyw`
      
      ### Dependencies

    draft: false
    prerelease: ${{ contains(github.ref, 'alpha') || contains(github.ref, 'beta') || contains(github.ref, 'rc') }}
```

#### Pattern 3: Generate Checksums
```yaml
- name: Generate checksums
  run: |
    Get-ChildItem *.zip | ForEach-Object {
      $hash = (Get-FileHash $_.Name -Algorithm SHA256).Hash
      "$hash  $($_.Name)" | Out-File -Append checksums.sha256
    }

- name: Upload with checksums
  uses: actions/upload-artifact@v4
  with:
    name: release-files
    path: |
      *.zip
      checksums.sha256
```

```python
import pytest
```

```python
from PySide6.QtCore import Qt, Signal, QPropertyAnimation
from PySide6.QtWidgets import QWidget, QVBoxLayout, QStackedWidget
from qfluentwidgets import (
    Dialog, PushButton, PrimaryPushButton,
    TitleLabel, BodyLabel, ProgressBar
)

class FluentWizardDialog(Dialog):
    """Custom wizard dialog using Fluent UI components."""
    
    # Signals
    page_changed = Signal(int)  # Emits new page index
    completed = Signal()        # Emits when wizard is completed
    cancelled = Signal()        # Emits when wizard is cancelled
    
    def __init__(self, parent=None):
        super().__init__("", "", parent)
        self.current_index = 0
        self.pages = []
        self._setup_ui()
    
    def _setup_ui(self):
        """Set up the wizard UI."""
        # Main layout
        self.mainLayout = QVBoxLayout(self.widget)
        
        # Title
        self.title_label = TitleLabel("Setup Wizard")
        self.mainLayout.addWidget(self.title_label)
        
        # Progress indicator
        self.progress_bar = ProgressBar()
        self.progress_bar.setRange(0, 100)
        self.mainLayout.addWidget(self.progress_bar)
        
        # Stacked widget for pages
        self.pages_widget = QStackedWidget()
        self.mainLayout.addWidget(self.pages_widget)
        
        # Buttons
        self._setup_buttons()
    
    def _setup_buttons(self):
        """Set up navigation buttons."""
        button_layout = QHBoxLayout()
        
        self.back_button = PushButton("Back")
        self.back_button.clicked.connect(self.back)
        self.back_button.setEnabled(False)
        
        self.next_button = PrimaryPushButton("Next")
        self.next_button.clicked.connect(self.next)
        
        self.finish_button = PrimaryPushButton("Finish")
        self.finish_button.clicked.connect(self.finish)
        self.finish_button.setVisible(False)
        
        self.cancel_button = PushButton("Cancel")
        self.cancel_button.clicked.connect(self.reject)
        
        button_layout.addWidget(self.back_button)
        button_layout.addStretch()
        button_layout.addWidget(self.cancel_button)
        button_layout.addWidget(self.next_button)
        button_layout.addWidget(self.finish_button)
        
        self.mainLayout.addLayout(button_layout)
    
    def add_page(self, page: QWidget, title: str):
        """Add a page to the wizard."""
        page.title = title
        self.pages.append(page)
        self.pages_widget.addWidget(page)
        self._update_progress()
    
    def next(self):
        """Go to next page."""
        if self.current_index < len(self.pages) - 1:
            # Validate current page
            current_page = self.pages[self.current_index]
            if hasattr(current_page, 'validate') and not current_page.validate():
                return
            
            self.current_index += 1
            self._update_page()
    
    def back(self):
        """Go to previous page."""
        if self.current_index > 0:
            self.current_index -= 1
            self._update_page()
    
    def finish(self):
        """Complete the wizard."""
        # Validate final page
        current_page = self.pages[self.current_index]
        if hasattr(current_page, 'validate') and not current_page.validate():
            return
        
        self.completed.emit()
        self.accept()
    
    def _update_page(self):
        """Update the current page display."""
        self.pages_widget.setCurrentIndex(self.current_index)
        
        # Update title
        current_page = self.pages[self.current_index]
        self.title_label.setText(current_page.title)
        
        # Update buttons
        self.back_button.setEnabled(self.current_index > 0)
        
        is_last_page = self.current_index == len(self.pages) - 1
        self.next_button.setVisible(not is_last_page)
        self.finish_button.setVisible(is_last_page)
        
        # Update progress
        self._update_progress()
        
        # Emit signal
        self.page_changed.emit(self.current_index)
        
        # Call page's on_show if it exists
        if hasattr(current_page, 'on_show'):
            current_page.on_show()
    
    def _update_progress(self):
        """Update progress bar."""
        if len(self.pages) > 0:
            progress = int((self.current_index / (len(self.pages) - 1)) * 100)
            self.progress_bar.setValue(progress)


class WizardPage(QWidget):
    """Base class for wizard pages."""
    
    def __init__(self, title: str, parent=None):
        super().__init__(parent)
        self.title = title
    
    def validate(self) -> bool:
        """
        Validate the page before moving to next.
        Override in subclass.
        """
        return True
    
    def on_show(self):
        """
        Called when page is shown.
        Override in subclass to perform initialization.
        """
        pass
```

```python
import json
from pathlib import Path
from typing import Any, Optional

class ConfigService:
    """Service for managing application configuration."""
    
    def __init__(self, config_path: Optional[Path] = None):
        if config_path is None:
            # Default to user's app data directory
            config_path = Path.home() / ".pymm" / "config.json"
        
        self.config_path = Path(config_path)
        self.config_path.parent.mkdir(parents=True, exist_ok=True)
        
        self.config = self._load()
    
    def _load(self) -> dict:
        """Load configuration from file."""
        if self.config_path.exists():
            try:
                with open(self.config_path, 'r', encoding='utf-8') as f:
                    return json.load(f)
            except (json.JSONDecodeError, IOError):
                return {}
        return {}
    
    def _save(self):
        """Save configuration to file."""
        try:
            with open(self.config_path, 'w', encoding='utf-8') as f:
                json.dump(self.config, f, indent=2, ensure_ascii=False)
        except IOError as e:
            print(f"Failed to save configuration: {e}")
    
    def get(self, key: str, default: Any = None) -> Any:
        """Get a configuration value."""
        keys = key.split('.')
        value = self.config
        
        for k in keys:
            if isinstance(value, dict) and k in value:
                value = value[k]
            else:
                return default
        
        return value
    
    def set(self, key: str, value: Any):
        """Set a configuration value."""
        keys = key.split('.')
        config = self.config
        
        for k in keys[:-1]:
            if k not in config:
                config[k] = {}
            config = config[k]
        
        config[keys[-1]] = value
        self._save()
    
    def should_show_first_run(self) -> bool:
        """Check if first-run dialog should be shown."""
        return self.get("ui.show_first_run_dialog", True)
    
    def mark_first_run_complete(self, dont_show_again: bool = False):
        """Mark first-run as complete."""
        self.set("setup.initial_setup_complete", True)
        if dont_show_again:
            self.set("ui.show_first_run_dialog", False)
```

```python
        dont_show = self.completion_page.should_hide_dialog()
        self.config_service.mark_first_run_complete(dont_show)
        
        # Emit completion signal
        self.setup_complete.emit(config_data)
        
        super().finish()
    
    @staticmethod
    def should_show(config_service: ConfigService) -> bool:
        """Check if first-run dialog should be shown."""
```

```python
class StorageSelectionPage(WizardPage):
    """Page for detecting and selecting storage locations."""
    
    storage_detected = Signal(list)  # Emits list of detected paths
    
    def __init__(self, parent=None):
        super().__init__("Storage Locations", parent)
        self.detected_paths = []
        self._setup_ui()
    
    def _setup_ui(self):
        layout = QVBoxLayout(self)
        
        # Instructions
        info_label = BodyLabel(
            "Select where your media files are stored.\n"
            "Click 'Detect' to automatically find common storage locations."
        )
        info_label.setWordWrap(True)
        layout.addWidget(info_label)
        
        # Detect button
        self.detect_button = PrimaryPushButton("Detect Storage Locations")
        self.detect_button.clicked.connect(self._detect_storage)
        layout.addWidget(self.detect_button)
        
        # List of detected locations
        self.storage_list = ListWidget()
        self.storage_list.setSelectionMode(ListWidget.MultiSelection)
        layout.addWidget(self.storage_list)
        
        # Manual add button
        self.add_button = PushButton("Add Custom Path...")
        self.add_button.clicked.connect(self._add_custom_path)
        layout.addWidget(self.add_button)
    
    def _detect_storage(self):
        """Detect storage locations."""
        # Show loading
        self.detect_button.setEnabled(False)
        self.detect_button.setText("Detecting...")
        
        # Perform detection (would be async in real implementation)
        from pymm.core.storage_detector import detect_storage_locations
        self.detected_paths = detect_storage_locations()
        
        # Update UI
        self.storage_list.clear()
        for path in self.detected_paths:
            item = ListWidgetItem(str(path))
            item.setData(Qt.UserRole, path)
            item.setSelected(True)  # Select by default
            self.storage_list.addItem(item)
        
        self.detect_button.setEnabled(True)
        self.detect_button.setText("Detect Storage Locations")
        
        # Emit signal
        self.storage_detected.emit(self.detected_paths)
    
    def _add_custom_path(self):
        """Add a custom storage path."""
        from qfluentwidgets import FolderDialog
        
        folder = FolderDialog.getExistingDirectory(
            self,
            "Select Storage Folder",
            ""
        )
        
        if folder:
            item = ListWidgetItem(folder)
            item.setData(Qt.UserRole, Path(folder))
            item.setSelected(True)
            self.storage_list.addItem(item)
    
    def get_selected_paths(self) -> list:
        """Get list of selected storage paths."""
        selected = []
        for i in range(self.storage_list.count()):
            item = self.storage_list.item(i)
            if item.isSelected():
                selected.append(item.data(Qt.UserRole))
        return selected
    
    def validate(self) -> bool:
        """Validate that at least one path is selected."""
        if not self.get_selected_paths():
            from qfluentwidgets import InfoBar, InfoBarPosition
            InfoBar.warning(
                title="No Storage Selected",
                content="Please select at least one storage location.",
                parent=self,
                position=InfoBarPosition.TOP
            )
            return False
        return True
    
    def on_show(self):
        """Auto-detect when page is shown."""
        if not self.detected_paths:
            self._detect_storage()
```

```python
class PluginConfigurationPage(WizardPage):
    """Page for initializing and configuring plugins."""
    
    plugins_initialized = Signal()
    
    def __init__(self, parent=None):
        super().__init__("Plugin Configuration", parent)
        self.plugin_widgets = {}
        self._setup_ui()
    
    def _setup_ui(self):
        layout = QVBoxLayout(self)
        
        # Info label
        info = BodyLabel("Enable the plugins you want to use:")
        layout.addWidget(info)
        
        # Scroll area for plugins
        from qfluentwidgets import ScrollArea
        scroll = ScrollArea()
        scroll.setWidgetResizable(True)
        
        plugins_container = QWidget()
        self.plugins_layout = QVBoxLayout(plugins_container)
        scroll.setWidget(plugins_container)
        
        layout.addWidget(scroll)
    
    def on_show(self):
        """Initialize plugins when page is shown."""
        from pymm.core.plugin_manager import PluginManager
        
        # Clear existing
        while self.plugins_layout.count():
            item = self.plugins_layout.takeAt(0)
            if item.widget():
                item.widget().deleteLater()
        
        # Load plugins
        plugin_manager = PluginManager()
        available_plugins = plugin_manager.get_available_plugins()
        
        for plugin_info in available_plugins:
            widget = PluginConfigWidget(plugin_info)
            self.plugin_widgets[plugin_info['id']] = widget
            self.plugins_layout.addWidget(widget)
        
        self.plugins_layout.addStretch()
        self.plugins_initialized.emit()
    
    def get_enabled_plugins(self) -> list:
        """Get list of enabled plugin IDs."""
        enabled = []
        for plugin_id, widget in self.plugin_widgets.items():
            if widget.is_enabled():
                enabled.append(plugin_id)
        return enabled
```

```python
    # Check if first-run dialog should be shown

        if dialog.exec() != dialog.Accepted:
            # User cancelled setup
            return 1
    
    # Show main window

    main_window.show()
    
    return app.exec()
```

## Summary and Recommendations

### setuptools-scm
- **Use simple configuration** with `setuptools-scm[simple]` for basic projects
- **Always use "v" prefix** for version tags (v1.0.0, v2.1.0, etc.)
- **Write version file** for portable distribution: `version_file = "src/pymm/_version.py"`
- **Fetch full Git history** in CI/CD: `fetch-depth: 0`

### GitHub Actions
- **Download embeddable Python** in workflow for portable builds
- **Configure site-packages** by editing `python*._pth` file
- **Freeze dependencies** using `pip freeze > requirements-frozen.txt`
- **Create archives** with version number in filename
- **Upload artifacts** to workflow and GitHub Releases

### pytest-qt
- **Use qtbot fixture** for all Qt tests
- **Organize tests** in parallel structure to source code
- **Test signals** using `qtbot.waitSignal()`
- **Prefer widget methods** over `qtbot` interactions (e.g., `setText()` over `keyClicks()`)
- **Mock external dependencies** to keep tests fast
- **Measure coverage** of GUI code with pytest-cov

### PySide6 Dialogs
- **Create custom wizard** using QStackedWidget and Fluent UI components
- **Persist settings** using ConfigService with JSON storage
- **Implement "don't show again"** with checkbox and config persistence
- **Structure wizard pages** as separate WizardPage subclasses
- **Validate pages** before allowing navigation
- **Emit signals** for completion and cancellation
